# 📢 راهنمای مدیریت تبلیغات TiksarVPN (از روی هاست)

این فایل توضیح می‌ده که برای **کنترل کاملِ تبلیغِ داخلِ اپ** روی هاستت چی باید بذاری.
با این روش، تبلیغ رو **هر وقت خواستی** روشن/خاموش یا عوض می‌کنی — **بدون هیچ بیلد و آپدیت جدیدی از اپ**.

---

## ۱) فایلی که باید روی هاست بذاری

اپ این آدرس ثابت رو می‌خونه (در کد: `ad_service.dart` → `_adUrl`):

```
https://up.tiksar.ir/up-tik/tiksar-ads.json
```

پس باید یه فایل به اسم **`tiksar-ads.json`** بسازی و توی پوشهٔ **`up-tik`** روی هاستت آپلود کنی
(همون پوشه‌ای که فایل چک‌آپدیت `tiksar-vpn.json` توش هست).

> ⚠️ نکته: خودِ فایلِ عکس یا ویدیو رو هم باید یه جای HTTPS آپلود کنی (مثلاً پوشهٔ `ads` روی همین هاست)
> و لینکش رو داخل `media_url` بذاری. مدیا **حتماً باید HTTPS باشه** (نه http).

---

## ۲) کلیدهای فایل JSON (معنی هر کدوم)

| کلید | نوع | معنی | مقدار مجاز |
|---|---|---|---|
| `enabled` | بولین | روشن/خاموش کردن کلِ تبلیغ | `true` = روشن، `false` = خاموش |
| `type` | متن | نوع تبلیغ | `"image"` (عکس) یا `"video"` (ویدیو) |
| `media_url` | متن | لینک مستقیمِ عکس یا ویدیو | یه URL کاملِ **HTTPS** |
| `click_url` | متن | لینکی که با کلیکِ کاربر روی تبلیغ باز می‌شه | هر URL (تلگرام، سایت، ...) — اگه خالی بذاری، کلیک کاری نمی‌کنه |
| `skip_after` | عدد | چند ثانیه بعد، دکمهٔ بستن (ضربدر ✕) فعال بشه | عددِ بین `1` تا `30` (بیشتر/کمتر بذاری، خودش clamp می‌شه) |
| `show_on` | متن | کجا نشون داده بشه | فعلاً فقط `"app_open"` (موقع باز شدن اپ) |

---

## ۳) نمونهٔ آماده — تبلیغِ ویدیویی (کپی کن، همینو آپلود کن)

```json
{
  "enabled": true,
  "type": "video",
  "media_url": "https://up.tiksar.ir/ads/promo.mp4",
  "click_url": "https://t.me/tiksar_vpn",
  "skip_after": 5,
  "show_on": "app_open"
}
```

## ۴) نمونهٔ آماده — تبلیغِ عکسی

```json
{
  "enabled": true,
  "type": "image",
  "media_url": "https://up.tiksar.ir/ads/banner.jpg",
  "click_url": "https://t.me/tiksar_vpn",
  "skip_after": 3,
  "show_on": "app_open"
}
```

## ۵) خاموش‌کردنِ کاملِ تبلیغ

هر وقت خواستی تبلیغ **اصلاً نشون داده نشه**، فقط کافیه `enabled` رو `false` کنی
(بقیه‌ی مقادیر مهم نیست):

```json
{
  "enabled": false,
  "type": "image",
  "media_url": "",
  "click_url": "",
  "skip_after": 5,
  "show_on": "app_open"
}
```

---

## ۶) جریان کار (چطوری تبلیغ رو عوض کنم؟)

۱. فایلِ عکس/ویدیوی جدید رو روی هاست آپلود کن (مثلاً `https://up.tiksar.ir/ads/promo2.mp4`).
۲. توی `tiksar-ads.json` مقدارِ `media_url` (و در صورت نیاز `click_url`) رو عوض کن.
۳. فایل `tiksar-ads.json` رو دوباره آپلود/ذخیره کن.
✅ تمام — کاربرها دفعهٔ بعد که اپ رو باز کنن، تبلیغِ جدید رو می‌بینن. (اپ فایل رو با no-cache می‌خونه، پس فوری اعمال می‌شه.)

---

## ۷) نکات و رفتار اپ (خوبه بدونی)

- تبلیغ **موقع باز شدن اپ** یک‌بار نشون داده می‌شه، **قبل از** دیالوگ چک‌آپدیت.
- تا وقتی `skip_after` ثانیه نگذشته، نه ضربدر هست نه دکمهٔ back سیستم اجازهٔ بستن می‌ده (دقیقاً مثل تبلیغ‌های گوگل).
- کلیک روی **هر جای تبلیغ** → لینکِ `click_url` رو در مرورگر باز می‌کنه.
- ویدیو به‌صورت **autoplay بی‌صدا** پخش می‌شه (سیاست اندروید)؛ کاربر با دکمهٔ صدا می‌تونه صدا رو روشن کنه.
- اگه فایل JSON نبود، خراب بود، یا مدیا لود نشد → اپ **هیچ‌وقت کرش نمی‌کنه**، فقط تبلیغ رو رد می‌کنه.
- ویدیوی پیشنهادی: کوتاه (۵ تا ۷ ثانیه) و کم‌حجم، چون باید موقع باز شدن اپ سریع لود بشه.

---

## ۸) چک‌لیست نهایی

- [ ] فایلِ مدیا (عکس/ویدیو) روی هاست با لینک **HTTPS** آپلود شد.
- [ ] فایل `tiksar-ads.json` ساخته و توی `up-tik/` آپلود شد.
- [ ] `enabled` روی `true` هست.
- [ ] `media_url` و `click_url` درست و کامل واردشده‌ن.
- [ ] لینک `https://up.tiksar.ir/up-tik/tiksar-ads.json` رو توی مرورگر باز کردی و فایل درست باز می‌شه.
